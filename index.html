Источник данных: OpenRCA Bank dataset, день

Что это: демонстрация классификации алертов по уровням L0 → L1 → L2 на реальных данных.

  • L0 — события: сырые алерты из трёх реальных источников датасета:
    • Prometheus пороговые алерты на metric_container.csv — порог = p95 нормального дня
    • AppMonitor алерты на metric_app.csv — sr < p05 baseline или mrt > 1.5× p95 baseline
    • Log Agent Drain3-шаблоны из log_service.csv
  • L1 — шаблоны: повторяющиеся trigger keys (`zbx-cpu`, `log-tpl-N`, `oim-...`)
  • L2 — классы проблем: 7 семантических групп под Bank (JVM heap & GC, Slow SQL, Disk IO, …)

AutoResolve%: серия событий считается самоликвидировавшейся, если в ней ≤ 5 событий ИЛИ длительность ≤ 10 минут — то есть алерт «потух сам» без вмешательства. Высокий % при большом потоке событий = устаревшее шумное правило: моргает и тушится без реальной причины. Серия = последовательность срабатываний шаблона на одном хосте с разрывом < 5 минут.

Дубли: события на одном host в окне 1 мин из ≥ 2 источников переклассифицируются в класс «Дубли из разных систем».

Дополнительные метрики шума (в drill-down):

  • Ev/host = events / unique_hosts — среднее число событий на один хост. Высокое = шум сосредоточен на 1-2 хостах (per-host issue).
  • Dedup = events / unique(host + 5-min bucket) — среднее число дубликатов на одну пару (host, 5 мин). Значение > 1 = повторяющийся шум за одно окно.
  • Burstiness = P95 / median количеств событий по 5-мин окнам (per host). Высокое = редкие, но плотные всплески.
  • Days active = days_active / total_days. 1.0 = постоянный шум, < 1 = эпизодический.

Drain3 cluster #N: Drain3 — наш собственный алгоритм кластеризации логов. Мы (AIOps) сами обрабатываем сырые логи через Drain3: он автоматически группирует похожие строки в шаблоны, заменяя переменные части (IP, числа, ID) на <*>. «Cluster #23» означает «23-й уникальный шаблон, найденный Drain3 в логах». Это порядковый номер, никакой семантики в нём нет. То есть: Log Agent → сырые строки → Drain3 кластеризует → L1-шаблон №23 → мы назначаем L2-класс по ключевым словам.

ServiceTest1..11: замаскированные имена сервисов из оригинального датасета OpenRCA Bank. Реальные имена заменены на абстрактные для конфиденциальности.
